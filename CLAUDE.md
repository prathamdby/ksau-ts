# ksau-ts

TypeScript + Bun rewrite of ksau-go. Faithful 1:1 port — no fixes, no improvements.

## Runtime

- Bun (not Node.js)
- `bun add` for dependencies, never `npm install`

## Key Conventions

- All byte counts use `bigint` (mirrors Go `int64`)
- All HTTP calls use native `fetch` with `AbortSignal.timeout(ms)`
- No test suite — matches ksau-go which has none
- No parallel chunk uploads — single-worker is intentional

## Commit Format

```
type: summary (≤50 chars)

- Body bullet one
- Body bullet two
```

Types: `feat` (port), `fix` (correction), `chore` (tooling/CI), `docs` (SPEC/README), `refactor`, `style`

## Build

```sh
bun run src/main.ts          # dev
make build                   # local compile
make build_gh_actions        # all 5 platform targets
```

## Source Layout

```
src/
  main.ts
  cmd/         root, upload, quota, refresh, listRemotes, version, help, utils
  azure/       client, config, upload, quota, hash, constants, types, item
  crypto/      algo, quickxorhash, placeholder
```

## Embedded Files

`src/crypto/privkey.pem` and `src/crypto/passphrase.txt` — gitignored, required at build time.

## Issue Tracking (Seeds)

<!-- seeds-onboard-v:1 -->

This project uses [Seeds](https://github.com/jayminwest/seeds) for git-native issue tracking.

**At the start of every session**, run:

```
sd prime
```

This injects session context: rules, command reference, and workflows.

**Quick reference:**

- `sd ready` — Find unblocked work
- `sd create --title "..." --type task --priority 2` — Create issue
- `sd update <id> --status in_progress` — Claim work
- `sd close <id>` — Complete work
- `sd sync` — Sync with git (run before pushing)

### Before You Finish

1. Close completed issues: `sd close <id>`
2. File issues for remaining work: `sd create --title "..."`
3. Sync and push: `sd sync && git push`

## Prompt Management (Canopy)

<!-- canopy-onboard-v:1 -->

This project uses [Canopy](https://github.com/jayminwest/canopy) for git-native prompt management.

**At the start of every session**, run:

```
cn prime
```

This injects prompt workflow context: commands, conventions, and common workflows.

**Quick reference:**

- `cn list` — List all prompts
- `cn render <name>` — View rendered prompt (resolves inheritance)
- `cn emit --all` — Render prompts to files
- `cn update <name>` — Update a prompt (creates new version)
- `cn sync` — Stage and commit .canopy/ changes

**Do not manually edit emitted files.** Use `cn update` to modify prompts, then `cn emit` to regenerate.
